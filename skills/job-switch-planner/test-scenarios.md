# Test Scenarios for job-switch-planner

## Scenario 1: 完整流程（happy path）

用户：5 年后端 Go 开发，期望转向云原生方向。

验证点：
- CP1 能否从对话中提取完整画像
- CP2 能否识别用户期望转型的技术栈
- CP3-4 能否采集并分析市场信息
- CP5 推荐的匹配度是否合理
- CP6 能否做 gap 分析
- CP7 计划是否针对 gap 设计

## Scenario 2: 简历粘贴输入

用户直接粘贴简历文本。

验证点：
- CP1 能否从非结构化文本中提取结构化信息
- 缺失字段是否主动追问

## Scenario 3: 中途回退

用户完成 CP5 后想修改 CP2 的期望薪资。

验证点：
- 回退到 CP2 后能否保留原有输入作为默认值
- 后续 CP3-CP5 数据是否标记为过时
- 重新执行后结果是否合理

## Scenario 4: 平台受限

某个招聘平台需要登录或反爬。

验证点：
- Playwright 访问受限时是否优雅跳过并记录
- 不影响其他平台的采集

## Scenario 5: 期望不合理

3 年经验用户期望薪资 100K。

验证点：
- CP2 是否能温和提示市场行情
- 是否给出合理参考区间
